# Review Queue Rules

Status: Review routing guidance for current and future evidence manifold work.

## Review Statuses

- source_backed
- needs_source_review
- unresolved
- placeholder
- out_of_scope
- accepted_with_caveat
- rejected_as_unsupported

## Routing Rules

| Input condition | Review route |
| --- | --- |
| official source with direct claim | review for source-backed status |
| third-party article | review with caveat |
| ambiguous relation | unresolved |
| missing source | needs_source_review |
| unsupported claim | rejected_as_unsupported |
| outside domain scope | out_of_scope |

## Status Guidance

### source_backed

Use when evidence directly supports the limited claim being made and a review decision documents the reason.

### needs_source_review

Use when a record, relationship, or claim is structurally useful but still lacks adequate source review.

### unresolved

Use when a claim or relation cannot yet be resolved from available evidence.

### placeholder

Use when a record exists to preserve structure or expected future coverage but should not be treated as supported.

### out_of_scope

Use when a claim may be interesting but does not belong in the current domain instance.

### accepted_with_caveat

Use when evidence supports a limited reading but requires an explicit caveat.

### rejected_as_unsupported

Use when reviewed evidence does not support the claim.

## Review Rules

- Official source with direct claim routes to source-backed review.
- Third-party article routes to review with caveat.
- Ambiguous relation routes to unresolved.
- Missing source routes to needs_source_review.
- Unsupported claim routes to rejected_as_unsupported.
- Outside-domain claim routes to out_of_scope.

## Non-Negotiable Boundary

Review status is not a product. Payment cannot buy verification status, source-backed status, badges, favorable treatment, changed evidence standards, review decisions, or inclusion.
